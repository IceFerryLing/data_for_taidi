# 基于事件挖掘的量化框架比赛算法说明

## 1. 参赛视角下的整体思路

本方案面向“根据事件挖掘构建量化投资框架”的比赛任务，核心目标不是直接做传统价格择时，而是把**事件文本信息、事件-公司关联图谱、个股历史行情**三类异构数据压缩成统一的周频决策样本，再用机器学习模型对“下一决策周哪些股票更值得配置”进行排序。

从参赛者视角看，这个算法解决的是一个典型的事件驱动选股问题：

1. 先识别某个事件会影响哪些公司。
2. 再评估这种影响的方向、强度、时效性和可交易性。
3. 最后结合个股已有价格状态与市场背景，预测下一个交易周的收益与上涨概率。

因此，这不是一个只看事件情绪的模型，也不是一个只看价格动量的模型，而是一个**“事件图谱驱动 + 价格状态修正 + 周频排序决策”**的量化框架。

---

## 2. 算法对应的数据输入

`pred_compact.py` 实际使用的核心输入包括：

- `events.csv`：事件主表，提供 `event_id`、`event_name`、`event_date` 等信息。
- `event_company_map.csv`：事件与上市公司映射表，提供 `relation_strength`、`relation_layer`、`relation_type` 等关联属性。
- `daily_kline_batches` 或 `price_panel.csv`：个股历史行情。
- `a_share_universe.csv`、`stock_industry_all.csv`：股票行业信息。
- `stock_basic_all.csv`：股票上市日期等基础信息。
- `hs300_constituents.csv`、`sz50_constituents.csv`、`zz500_constituents.csv`：指数成分信息，可选使用。

这套输入决定了模型本质上是在回答如下问题：

> 对于某个周二形成的候选股票集合，在接下来这一周里，哪些股票最可能因为近期事件驱动取得更好的收益表现？

---

## 3. 交易问题的数学定义

### 3.1 决策时点

代码把每周二定义为一个统一的决策时点 `decision_tuesday`。  
这是因为事件发生后需要一定的传播和交易消化时间，同时比赛输出是周频持仓决策，周二作为统一对齐点能兼顾时效性和样本稳定性。

对任意事件日期 $t_e$，算法用如下规则映射到决策周二：

- 如果事件发生在周一，则归入本周周二。
- 如果事件发生在周二及之后，则归入下周周二。

记映射函数为：

```math
T(e) = \text{next\_decision\_tuesday}(t_e)
```

这样，每条“事件-股票”关系都会被归属到一个明确的周频决策截面。

### 3.2 监督学习标签

模型不是预测长期收益，而是预测从周二附近建仓到周末平仓这一周的收益。

代码中的标签定义为：

- 买入价：该周从周二开始可交易日的第一个 `open`
- 卖出价：该周到周五为止最后一个 `close`

于是单周目标收益率为：

```math
y_{i,t} = \frac{P^{\text{sell}}_{i,t} - P^{\text{buy}}_{i,t}}{P^{\text{buy}}_{i,t}}
```

其中：

- $i$ 表示股票
- $t$ 表示决策周二

为了减少极端行情对树模型的扰动，收益率被截断到：

```math
y_{i,t} \in [-0.5, 0.5]
```

同时还定义分类标签：

```math
z_{i,t} = \mathbf{1}(y_{i,t} > 0)
```

即：下一周是否上涨。

这意味着本算法是一个**回归 + 分类并行建模**的双目标框架。

## 4. 事件驱动样本如何构建

### 4.1 从事件图谱到股票候选

算法首先把 `event_company_map.csv` 与 `events.csv` 按 `event_id` 合并，得到“某事件影响某股票”的基础样本。每一行可以理解为一条边：

```math
e \rightarrow i
```

其中边上携带的信息包括：

- 关联强度 `relation_strength`
- 关联层级 `relation_layer`（直接/间接）
- 关联类型 `relation_type`
- 事件强度 `intensity_score`
- 驱动类型、影响周期、可预期性等描述变量

### 4.2 事件时效衰减

事件并不是越早越好。距离决策时点越远，交易价值通常越弱，所以代码引入了半衰期思想。

先定义事件距决策周二的天数：

```math
\Delta d = \min(\max(T(e)-t_e, 1), 30)
```

再定义衰减系数：

```math
\text{decay} = \exp\left(-\ln(2)\cdot \frac{\Delta d}{7}\right)
```

其含义是：**每经过 7 天，事件影响力大约衰减一半**。  
这是一种非常适合事件交易的权重设计，因为它符合“短期事件 alpha 快速衰减”的市场经验。

### 4.3 直接关系加权

代码区分“直接关联”和“间接关联”。  
若事件与股票是直接映射关系，权重额外乘以 $1.5$；若不是，则不加成。

记：

```math
\text{direct} \in \{0,1\}
```

则直接关系调整因子为：

```math
1 + 0.5 \cdot \text{direct}
```

### 4.4 事件强度与关联强度融合

最终单条事件-股票边的基础权重定义为：

```math
w_{e,i}
= \max(\text{relation\_strength}, 0.05)
\cdot \left(1+\frac{\text{intensity\_score}}{5}\right)
\cdot \text{decay}
\cdot (1+0.5\cdot \text{direct})
```

这个公式是整个框架的关键之一。它把四个核心维度融合到同一个事件影响强度里：

- `relation_strength`：事件与公司的结构性关联程度
- `intensity_score`：事件本身的强弱
- `decay`：时间衰减
- `direct`：是否为直接映射

其中 `max(relation_strength, 0.05)` 的设计是为了避免零强度关系在聚合时被完全抹掉，也避免树模型看到过多纯零特征。

## 5. 事件方向识别的数学含义

代码没有使用复杂的 NLP 深度模型，而是采用了**高可解释、低过拟合风险的规则化事件方向打分**。

### 5.1 事件正负方向

算法根据 `event_name`、`relation_type`、`note` 中是否出现典型利多/利空关键词，构造事件方向：

```math
\text{sign}_{e,i} \in \{-1,0,1\}
```

其中：

- 利多事件记为 `+1`
- 利空事件记为 `-1`
- 中性或无法判断记为 `0`

并进一步衍生：

```math
\text{positive\_event} = \mathbf{1}(\text{sign}_{e,i} > 0)
```

```math
\text{negative\_event} = \mathbf{1}(\text{sign}_{e,i} < 0)
```

这种做法的优点是：

- 对比赛型数据更稳健
- 可解释性强
- 在样本量有限时比复杂文本模型更不容易过拟合

## 6. 从多事件到单股票周频特征

一个股票在同一决策周可能对应多个事件，因此必须从“边级样本”聚合为“股票-周样本”。

设某个决策周 $t$ 上，股票 $i$ 对应的事件集合为 $E_{i,t}$。

算法围绕这个集合构造了以下几类聚合特征。

### 6.1 事件数量特征

```math
\text{event\_count}_{i,t} = |E_{i,t}|
```

```math
\text{unique\_event\_count}_{i,t} = \text{去重后的事件数量}
```

并统计正向、负向事件数量：

```math
\text{positive\_event\_count}_{i,t} = \sum_{e\in E_{i,t}} \mathbf{1}(\text{sign}_{e,i}>0)
```

```math
\text{negative\_event\_count}_{i,t} = \sum_{e\in E_{i,t}} \mathbf{1}(\text{sign}_{e,i}<0)
```

这些特征反映的是**某只股票在当前周的事件拥挤度与方向分布**。

### 6.2 权重统计特征

对上一节定义的事件边权重 $w_{e,i}$，做如下聚合：

```math
\text{row\_weight\_sum}_{i,t} = \sum_{e\in E_{i,t}} w_{e,i}
```

```math
\text{row\_weight\_max}_{i,t} = \max_{e\in E_{i,t}} w_{e,i}
```

```math
\text{row\_weight\_mean}_{i,t} = \frac{1}{|E_{i,t}|}\sum_{e\in E_{i,t}} w_{e,i}
```

其数学含义分别对应：

- `sum`：本周累计事件驱动力
- `max`：最强单一事件冲击
- `mean`：平均事件质量

### 6.3 关联强度与事件强度统计

分别对 $\text{relation\_strength}$ 和 $\text{intensity\_score}$ 取均值、最大值：

```math
\text{relation\_strength\_mean},\ \text{relation\_strength\_max}
```

```math
\text{intensity\_score\_mean},\ \text{intensity\_score\_max}
```

它们用于衡量“事件是否真的强相关”和“事件本身是否足够强”。

### 6.4 事件方向统计

```math
\text{event\_sign\_sum}_{i,t} = \sum_{e\in E_{i,t}} \text{sign}_{e,i}
```

```math
\text{event\_sign\_mean}_{i,t} = \frac{1}{|E_{i,t}|}\sum_{e\in E_{i,t}} \text{sign}_{e,i}
```

该组特征反映了**事件簇整体偏利多还是偏利空**。

### 6.5 时效特征

对事件到决策日的时间差做聚合：

```math
\text{decision\_gap\_days\_min},\ \text{decision\_gap\_days\_mean}
```

```math
\text{decay\_to\_decision\_max},\ \text{decay\_to\_decision\_mean}
```

这组变量让模型知道：

- 当前事件是刚发生，还是已经发酵多日
- 是单一新事件主导，还是多条旧事件叠加

### 6.6 关系层级特征

```math
\text{direct\_event\_share}_{i,t} = \frac{\sum \mathbf{1}(\text{direct}=1)}{|E_{i,t}|}
```

```math
\text{indirect\_event\_share}_{i,t} = \frac{\sum \mathbf{1}(\text{indirect}=1)}{|E_{i,t}|}
```

并统计其数量。  
其目的是让模型区分“核心受益标的”和“外围映射标的”。

## 7. 事件类别压缩：把文本属性转成可学习信号

代码没有把事件类别当作高维 one-hot，而是做了**权重聚合后的低维类别统计**，这是防止过拟合的关键设计。

### 7.1 驱动类型桶

事件被压缩到以下几类：

- `company`
- `industry`
- `macro`
- `geo`
- `policy`
- `other`

然后按事件权重求和，例如：

```math
\text{driver\_w\_macro}(i,t) = \sum_{e\in E_{i,t},\ \text{driver}(e)=\text{macro}} w_{e,i}
```

这比简单统计个数更强，因为它同时纳入了强度、时效和直接性。

### 7.2 影响周期桶

影响周期被映射为：

- `pulse`
- `mid`
- `long`
- `other`

同样定义加权和：

```math
\text{impact\_w\_pulse},\ \text{impact\_w\_mid},\ \text{impact\_w\_long}
```

这让模型能够识别：

- 短线脉冲型事件
- 中期驱动型事件
- 长尾逻辑型事件

### 7.3 可预期性桶

可预期性被压缩成：

- `sudden`
- `pre`
- `other`

并构造：

```math
\text{predict\_w\_sudden},\ \text{predict\_w\_pre}
```

这部分反映的是**事件是否容易被市场提前定价**。  
一般来说，突发事件更可能带来超额收益，而预披露事件更容易被提前消化。

---

## 8. 价格与市场状态特征

事件并不总能转化为收益，必须结合个股原有走势判断其承接能力，因此算法加入了紧凑的价格状态特征。

### 8.1 动量特征

定义 5 日和 20 日收盘动量：

```math
\text{close\_mom\_5} = \frac{\text{Close}_t}{\text{Close}_{t-5}} - 1
```

```math
\text{close\_mom\_20} = \frac{\text{Close}_t}{\text{Close}_{t-20}} - 1
```

其含义是：

- 5 日动量代表短期情绪和资金共振
- 20 日动量代表中短期趋势背景

### 8.2 波动率特征

先定义日收益率：

```math
r_t = \frac{\text{Close}_t}{\text{Close}_{t-1}} - 1
```

再计算滚动标准差：

```math
\text{volatility\_5} = \text{std}(r_{t-4:t})
```

```math
\text{volatility\_20} = \text{std}(r_{t-19:t})
```

波动率本质上衡量的是不确定性。  
同样的事件强度，在高波动股票和低波动股票上的收益兑现效率可能完全不同。

### 8.3 成交额与成交量活跃度

代码构造了：

```math
\text{amount\_ratio}_{5,20} = \frac{MA_5(\text{amount})}{MA_{20}(\text{amount})}
```

```math
\text{volume\_ratio}_{5,20} = \frac{MA_5(\text{volume})}{MA_{20}(\text{volume})}
```

它们用于判断事件发生前后资金是否已经开始异动。  
若该比值大于 1，意味着近期交易活跃度高于中期均值，可能说明事件正在被资金关注。

### 8.4 最近一日涨跌幅

```math
\text{last\_pctchg}
```

这是一个很轻量但有效的短期状态变量，用于表示事件周前最后一个交易日的价格反馈。

## 9. 横截面背景特征

### 9.1 行业匹配

如果事件所属行业与股票行业一致，则：

```math
\text{industry\_match} = 1
```

否则为 0。

之后对同周所有事件边取均值和最大值。  
这个变量的含义是：**该股票是否属于事件逻辑的主战场，而不是外围映射。**

### 9.2 上市时长

```math
\text{listed\_days} = \text{decision\_tuesday} - \text{ipo\_date}
```

并构造：

- `is_recent_ipo_180d`
- `is_recent_ipo_365d`

新股常常有更高弹性，也有更高噪声，因此需要显式纳入模型。

### 9.3 指数成分

代码支持加入：

- `is_hs300`
- `is_sz50`
- `is_zz500`

但默认关闭。  
这体现了一个很稳健的建模思想：**在事件驱动策略里，指数风格暴露不是必须信号，只有在验证有效时才启用。**

## 10. 上周收益上下文特征

为了避免模型只看当前事件，不看市场记忆，代码加入了上一周的个股和市场背景。

### 10.1 个股上周收益

$$
\text{prev\_week\_return}_{i,t} = y_{i,t-1}
$$

```math
\text{prev\_week\_up}_{i,t} = \mathbf{1}(y_{i,t-1} > 0)
```

这表示该股票此前是否已经开始被资金抢跑。

### 10.2 市场整体上周状态

对所有股票的上周收益做截面聚合：

```math
\text{market\_prev\_week\_mean}_t = \frac{1}{N_t}\sum_i y_{i,t-1}
```

```math
\text{market\_prev\_week\_std}_t = \text{std}_i(y_{i,t-1})
```

```math
\text{market\_prev\_week\_up\_ratio}_t = \frac{1}{N_t}\sum_i \mathbf{1}(y_{i,t-1} > 0)
```

这组变量是一个轻量的市场 regime 代理：

- 均值高：环境偏暖
- 标准差高：分化加剧
- 上涨比例高：风险偏好提升

## 11. 为什么使用压缩特征

代码中的类名就叫 `EventGraphPriceCompactStrategy`，说明策略设计强调“压缩特征、避免过拟合”。

比赛环境下，最常见的问题不是模型不够复杂，而是：

- 事件样本跨度有限
- 文本标签质量不完全稳定
- 股票截面虽然多，但真正有效样本周数有限

因此，本方案特意不做超高维文本向量、不做复杂图神经网络，而是把所有事件信息压缩为几十个解释清晰的统计量。  
这相当于在建模前先做一层有金融含义的结构化归纳，提升泛化能力。

## 12. 预测模型：回归与分类双模型并行

### 12.1 回归模型

回归器使用 `XGBoost Regressor`，目标是预测下一周收益率：

```math
\hat{y}_{i,t} = f_{\text{reg}}(x_{i,t})
```

其中 $x_{i,t}$ 为前文构造的压缩特征向量。

### 12.2 分类模型

分类器使用 `XGBoost Classifier`，目标是预测下一周上涨概率：

```math
\hat{p}_{i,t} = P(y_{i,t} > 0 \mid x_{i,t}) = f_{\text{cls}}(x_{i,t})
```

### 12.3 为什么双模型比单模型更合理

因为在量化决策里，收益预测和方向预测并不完全等价：

- 回归更关注幅度
- 分类更关注胜率

如果只看收益预测，模型可能偏好高波动股票；  
如果只看上涨概率，模型又可能忽略收益空间。

双模型并行相当于把交易决策拆成两个维度：

- 这只股票涨的概率高不高？
- 如果涨，能涨多少？

这非常符合比赛场景下的选股逻辑。

### 12.4 模型参数为何偏保守

代码中设置：

- `n_estimators = 140`
- `learning_rate = 0.035`
- `max_depth = 3`
- `min_child_weight = 8`
- `subsample = 0.72`
- `colsample_bytree = 0.72`
- `reg_alpha = 0.8`
- `reg_lambda = 3.0`
- `gamma = 0.2`

这些参数共同体现的思想是：

- 浅树：限制复杂度
- 小学习率：提高稳定性
- 较强正则：抑制噪声特征
- 行列采样：提升泛化

本质上，这是一套偏“比赛稳健解”的参数，而不是追求训练集拟合极致的参数。

## 13. 时序验证方式

算法没有随机打乱样本，而是严格按照时间做 walk-forward 验证。

设目标预测周为 $T$，则验证集取历史最近若干周，每次按如下方式滚动：

1. 用某个验证周之前的所有历史周训练模型。
2. 在该验证周上预测。
3. 把多个验证周的结果汇总。

这比随机交叉验证更符合实盘和比赛要求，因为它避免了未来信息泄露。

### 13.1 验证指标

代码使用了：

- `AUC`：判断上涨概率排序能力
- `R2`：衡量收益回归拟合度
- `RankIC`：预测收益与真实收益的相关性
- `Hit Rate`：上涨方向命中率
- `Top3 Mean Return`：每周选前 3 名后的平均收益

其中最贴近最终交易目标的是：

```math
\text{Top3 Mean Return}
```

因为比赛最终输出并不是全市场收益预测表，而是有限数量标的的资金分配结果。

## 14. 最终选股打分函数

在正式预测阶段，模型先得到：

- 回归预测收益 $\hat{y}_{i,t}$
- 分类上涨概率 $\hat{p}_{i,t}$

代码再定义综合得分 `selection_score`：

```math
\text{score}_{i,t}
= 0.58 \cdot \hat{p}_{i,t}
+ 0.22 \cdot \frac{\hat{y}_{i,t}}{\text{scale}_t}
+ 0.12 \cdot \text{weight\_norm}_{i,t}
+ 0.08 \cdot \text{prev\_week\_return}_{i,t}
```

其中：

```math
\text{scale}_t = \text{median}(|\hat{y}_{i,t}|) + 10^{-6}
```

```math
\text{weight\_norm}_{i,t} = \frac{\text{row\_weight\_sum}_{i,t}}{\max_j \text{row\_weight\_sum}_{j,t} + 10^{-6}}
```

这个综合得分的金融解释是：

- `0.58 * prob_up`：优先保证胜率
- `0.22 * pred_ret/scale`：兼顾收益空间
- `0.12 * weight_norm`：强化事件驱动主线
- `0.08 * prev_week_return`：承认短期延续性

这说明本算法不是让模型自己完全决定排序，而是在机器学习输出之上加入了交易层面的二次结构化决策。

## 15. 选股过滤与资金分配

### 15.1 过滤逻辑

先筛选：

```math
\hat{p}_{i,t} \ge 0.5
```

即至少上涨概率不低于 50%。  
如果筛完为空，则回退为直接按综合得分取前 `TopK`，这里默认 `K=3`。

### 15.2 资金权重

对入选股票的综合分做 softmax 分配：

```math
w_i = \frac{e^{\text{score}_i - \max(\text{score})}}{\sum_j e^{\text{score}_j - \max(\text{score})}}
```

其优点是：

- 分数高者获得更高仓位
- 仓位和严格等于 1
- 不会因为单个异常大分数导致权重爆炸

如果 softmax 失效，则回退为经验权重 `[0.5, 0.3, 0.2]`。

因此最终输出的不是简单股票名单，而是：

- 事件名称
- 股票代码
- 资金比例

这与比赛要求的可执行投资方案是一致的。

## 16. 从数学原理看，这个框架为什么有效

我认为该算法的有效性主要来自以下四点。

### 16.1 它把事件信息转成了“可交易强度”

很多事件挖掘方案止步于“识别事件是什么”，但本算法进一步回答了：

- 事件离交易时点多近？
- 与公司关系多直接？
- 事件本身多强？
- 是否属于主逻辑方向？

于是事件不再是文本标签，而是一个可量化的影响权重。

### 16.2 它把图谱关系做了聚合压缩

事件与公司不是一对一，而是一对多、多对多。  
代码通过周频聚合，把图谱关系从复杂网络压缩成适合监督学习的特征向量，避免了图结构直接建模在小样本比赛中的不稳定性。

### 16.3 它承认价格先于事件交易

很多事件策略忽略一个现实：市场常常会抢跑。  
加入动量、波动率、量价活跃度、上周收益等特征之后，模型能区分：

- 事件刚启动、尚未被交易
- 事件已被充分演绎、继续追高风险大

### 16.4 它把“预测”转成了“排序决策”

比赛最终比拼的不是某个单点预测值，而是选股结果。  
所以代码最终优化的是一套排序与配置机制，而不是单纯追求回归误差最小。这一点非常符合量化比赛的实战要求。

## 17. 算法流程总结

完整流程可以概括为：

1. 读取事件表、事件-公司图谱、个股行情和静态信息。
2. 把事件日期映射到统一的周二决策时点。
3. 计算下周持有期收益，构造监督学习标签。
4. 对每条事件-股票边计算事件权重、方向、时效、直接性等属性。
5. 将同一股票在同一决策周的多条事件聚合成紧凑特征。
6. 拼接价格动量、波动率、量价活跃度、上市时长、行业匹配、上周收益等辅助特征。
7. 用 XGBoost 回归器预测收益，用 XGBoost 分类器预测上涨概率。
8. 通过 walk-forward 时序验证评估模型稳定性。
9. 按综合得分排序，筛出 Top3 候选股票。
10. 用 softmax 做资金分配，输出最终投资方案。

## 18. 该方案的优点与局限

### 18.1 优点

- 可解释性强，便于比赛答辩和方案展示。
- 充分利用了事件、图谱、价格三类信息。
- 使用周频压缩特征，泛化能力较强。
- 双模型结构同时兼顾胜率与收益空间。
- 排序与资金分配逻辑完整，可直接落地。

### 18.2 局限

- 事件正负方向仍采用规则法，语义刻画有限。
- 没有显式建模事件之间的传播链与时序联动。
- 周频框架对极短线事件的捕捉可能不如日频策略。
- 资金分配只基于分数 softmax，尚未加入风险预算约束。

## 19. 结论

从参赛方案的角度，这个算法的核心价值在于：  
它不是把“事件挖掘”停留在文本识别层面，而是进一步构造出一套**事件影响量化、图谱关系聚合、价格状态修正、双模型预测、排序配置输出**的完整闭环。

可以把它概括为一句话：

> 先用事件图谱判断“谁会受影响”，再用时间衰减和价格状态判断“影响能否兑现”，最后用双模型排序决定“买谁、买多少”。

这正是一套适合事件驱动量化比赛提交的完整框架。
